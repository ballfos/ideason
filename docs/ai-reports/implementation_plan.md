
# 包括的なテスト計画 (E2Eテスト) の導入

システムの主要なユーザーフローを網羅し、品質を担保するためのE2E（エンドツーエンド）テストを導入します。テストの記述には Gherkin (Cucumber) 形式の `.feature` ファイルを使用し、Playwright と TypeScript で実装します。

## 1. 必要なテスト項目のリストアップ

1. **認証 (Authentication)**:
    - ログイン画面が正しく表示されること。
    - Googleログインボタンが機能すること（Firebase Authのモックまたはテスト用アカウント使用）。
    - ログイン後、ホームページ（あいでぃあ村）へ遷移すること。
    - ログアウト後、ログイン画面へ戻ること。

2. **トーク作成フロー (Talk Creation)**:
    - 新しいトーク作成画面でテーマとメンバーを入力し、作成できること。
    - テーマが未入力の場合にバリデーションエラーが表示されること。
    - 作成成功後、対応するトークIDのチャット画面へ遷移すること。

3. **チャット & AI応答 (Chat & AI Response)**:
    - ユーザーがメッセージを送信し、チャット履歴に追加されること。
    - AIエージェントが順次応答し、ストリーミング形式で表示されること。
    - 特定のキーワード（スタンプIDなど）が入力された際、画像スタンプとして表示されること。

4. **アイデアマップ (Idea Map)**:
    - チャットでの議論が進むと、アイデアマップ（ホワイトボード）にノードが自動生成されること。
    - マップ内のノードをクリックして詳細を確認できること。

5. **メンバー管理 (Member Management)**:
    - トーク詳細のメニューから、現在のメンバー一覧を確認できること。
    - メンバーの追加・削除・編集（名前、役割、アイコン）が即座に反映されること。

6. **ホーム画面 & ナビゲーション (Home & Navigation)**:
    - ホーム画面の鹿画像がランダムに表示されること。
    - 「最近のトピック」一覧から既存のトークへ遷移できること。

## 2. Gherkinテストの代替案

Playwrightでの直接的な実装のほかに、以下の方法も検討可能です。

### A. Cucumber.js + Playwright (標準的)
- **特徴**: Gherkinの標準的な実行エンジンであるCucumber.jsを使用。
- **メリット**: 非常に多機能で、複雑なシナリオ管理に適している。
- **デメリット**: 設定がやや複雑になる可能性がある。

### B. vitest-cucumber (軽量・高速)
- **特徴**: 既存のVitest環境内でGherkinを動作させる。
- **メリット**: `npm run test` と統合でき、実行が非常に速い。コンポーネントレベルのテストにも強い。
- **デメリット**: 実際のブラウザ操作（E2E）としては、Playwrightほど詳細な検証ができない場合がある。
